# Gamification Design

mycraft의 게임 메커닉을 설계한다. **장식이 아니라 행동 변화 도구**로서.

## 핵심 질문 (모든 메커닉 결정 전 먼저)

1. 이 메커닉이 사용자의 **실제 행동**을 더 좋은 방향으로 끌어당기는가?
2. 사용자가 시스템을 **gaming**(편법으로 보상 따먹기)할 수 있는가? 그렇다면 어떻게 막을 것인가?
3. 1주 후, 1달 후에도 **동기 부여 효과가 유지**되는가? 아니면 일시적인가?

이 질문에 명확한 답이 없으면 메커닉을 채택하지 않는다.

## 메커닉 카테고리

### A. XP / 레벨

XP는 **노력의 단위**다. 결과가 아닌 행동을 보상한다 (예: "운동 1회 = 10 XP", "운동 30분 = 25 XP").

**XP 공식 결정 시 고려:**
- 활동 강도 차이 반영 (단순 빈도 vs 시간 vs 강도).
- 일일/주간 상한 (번아웃 방지).
- 스트릭 보너스 (연속 수행에 대한 가산).

**레벨 곡선 옵션:**

| 곡선 | 공식 예시 | 특성 |
|------|---------|-----|
| 선형 | `lvl = floor(xp / 100)` | 초기 빠름, 후기 단조로움 |
| 제곱 | `xp_to_next = lvl^2 * 50` | 초기 빠름, 점진적 둔화 (RPG 표준) |
| 로그 | `lvl = floor(log2(xp/50 + 1))` | 초반 빠르고 후반 매우 천천히 |

**기본 추천:** 제곱 곡선. 초기 보상감 + 후기 도전감 균형.

### B. 보상 (Reward)

보상은 **외재적**(배지, 칭호, 시각 효과)과 **내재적**(자기 기록 비교, 인사이트 제공)이 있다.

mycraft는 1인 사용자 제품이므로 **내재적 보상 우선**:
- 새 레벨 도달 시 누적 통계 시각화 ("당신은 지난 30일간 X km 달렸습니다").
- 스트릭 깨질 때 단순 알림 대신 회복 제안 ("어제 쉬었네요. 오늘 짧게 시작해볼까요?").

외재적 보상은 **수집 욕구를 자극**하지만 남발 시 의미 인플레이션. 한정 + 의미 부여 (예: "30일 연속 명상 — 이 배지는 6개월간 1회만 획득 가능").

### C. 퀘스트

퀘스트 = **명시된 목표 + 기한 + 보상**.

3단계 권장:
- **데일리 퀘스트** — 매일 자정 리셋. 작고 달성 가능한 목표.
- **위클리 퀘스트** — 주 단위. Sage의 주간 리뷰가 다음 주 위클리 퀘스트를 추천.
- **에픽 퀘스트** — 월 단위 / 사용자 자율. 큰 목표.

각 퀘스트는 **스킵 가능**. 강제하면 부담 → 사용 중단.

### D. 스탯 (Stat)

스탯 = **사용자의 다면을 보여주는 수치 모음**.

mycraft 추천 스탯 카테고리 (사용자가 커스터마이즈 가능):
- **체력 (Vitality)** — 운동/수면 관련.
- **지성 (Intelligence)** — 학습/독서 관련.
- **집중 (Focus)** — 명상/딥워크 관련.
- **사교 (Social)** — 소통 활동 관련.

**원칙:** 스탯은 5~7개 이내. 너무 많으면 사용자가 추적 못 함. 카테고리당 최소 1개 활동이 매핑되어야 의미 있음.

## 산출물 형식

기능 정의서(`_workspace/quest_{topic}.md`)에 다음 표 포함:

```markdown
## 게임 메커닉 명세

### XP 공식
- {활동}: {XP 양} (조건: {조건})

### 레벨 곡선
- 함수: {공식}
- 1→2: {필요 XP}, 5→6: {필요 XP}, 10→11: {필요 XP}

### 보상
| 트리거 | 보상 종류 | 내용 |
|-------|---------|-----|
| 레벨업 | 내재적 | 누적 통계 카드 |
| 7일 스트릭 | 외재적 | "Iron Will" 배지 |

### 퀘스트
| 등급 | 예시 | 보상 |
|------|------|------|
| 데일리 | "오늘 운동 1회" | 10 XP |
| 위클리 | "이번 주 운동 4회" | 50 XP + 배지 |

### 스탯 영향
- {활동} → {스탯}: +{값}
```

## 안티패턴 (피할 것)

- **인플레이션 보상** — 모든 행동에 큰 XP/배지를 줘서 의미가 희석되는 것.
- **강제 스트릭** — 한 번 깨면 영구 손실. 사용자가 단기 휴식조차 두려워하게 됨.
- **숨겨진 공식** — 사용자가 왜 이 보상을 받았는지 모르는 것. 투명하게 공개.
- **장식적 게이미피케이션** — 행동 변화 없이 화려한 UI만. 검증 필요 시 1주 시범 운영 후 사용자 행동 변화 측정.

## 검증 (Sage와 협업)

새 메커닉 도입 후 1주 / 1달 시점에 Sage가 사용자 행동 변화를 측정한다:
- 메커닉 도입 전후 활동 빈도/지속시간 비교.
- 사용자가 메커닉을 의식적으로 회피/이용했는지.
- 부담감 증가 신호 (활동량 급감, 기록 누락).

부정적이면 메커닉을 수정/제거. AGENTS.md 변경 이력에 기록.
