# Design: 시즌 챌린지 운영 체계

> 작성: Artisan | 2026-05-06 | 입력: `quest_season_challenge_system.md` | 다음 페르소나: Dungeon Architect

## 0. 디자인 방향

현재 화면은 "활동 카드 목록"이다. 이 구조는 명확하지만 너무 흔하다. 새 방향은 "시즌 커맨드 센터"다.

사용자가 앱을 열었을 때 물어보는 것은 "무엇을 기록할까?"가 아니다.

> 지금 시즌에서 오늘 무엇을 확보해야 하지?

## 1. 정보 구조

### 최상위 화면

`/season` 또는 루트 홈이 중심이다.

```
Season Command
  - 현재 시즌 상태
  - 이번 주 작전
  - 오늘의 미션
  - 시즌 지도
  - 빠른 기록
```

`/today`는 남겨도 되지만 첫 화면이 아니다. Today는 Season Command 안의 "오늘 미션 상세"로 흡수한다.

### 화면 우선순위

1. 오늘 해야 할 미션
2. 이번 주 작전의 판도
3. 시즌 전체 진행
4. 기록 상세와 카테고리 관리

## 2. 모바일 와이어프레임

```

│ │ 딥워크 60분             │ │
│ │ [시작] [+15분]          │ │
│ └─────────────────────────┘ │
├─────────────────────────────┤
│ Week 2: 압축 작전           │
│ ███░░ 3/5 구역 확보         │

├─────────────────────────────┤
```

특징:

- 상단은 레벨보다 시즌 상태가 먼저다.
- 미션 카드는 2~3개만 보인다.
- 버튼은 "기록"이 아니라 "완료", "시작", "대체"처럼 행동 지향이다.
- 기존 카테고리 추가 버튼은 첫 화면에서 숨긴다.

## 4. 컴포넌트

### `SeasonHeader`

역할: 현재 시즌을 앱의 첫 신호로 만든다.

내용:

- 시즌 번호 또는 이름
- 남은 일수
- 주 스탯
- 시즌 승리 조건

상태:

- no-season: 새 시즌 시작 CTA
- active: 진행 상태
- ending-soon: 3일 이하 남음
- ended: 시즌 리뷰 CTA

### `MissionDeck`

역할: 오늘 해야 할 일을 카드 2~3장으로 압축한다.

카드 액션:

- 완료
- 시작
- 수치 추가
- 대체 수행
- 보류

원칙:

- 기본 완료는 1탭이다.
- 수치 입력 필드는 기본으로 숨긴다.
- "메모"는 카드 완료 후 접히는 추가 입력으로만 둔다.

### `SeasonMap`

역할: 월/주/일 계획이 연결되어 있음을 보여준다.

표현:

- 월간 = 지도 전체
- 주간 = 구역
- 일간 = 구역 안의 작은 노드

상태:

- clear
- active
- weak
- locked
- failed-but-adapted

### `EpisodePanel`

역할: 이번 주 작전을 보여준다.

내용:

- 이번 주 핵심 챌린지
- 보조 챌린지
- 버려도 되는 행동
- 위험 신호
- 다음 리뷰 시점

### `QuickLogDock`

역할: 기존 카테고리 기록을 숨기되 버리지 않는다.

동작:

- 화면 하단 또는 우측에 작게 둔다.
- 누르면 기존 카테고리 입력이 열린다.
- 기본 흐름은 미션 완료를 우선한다.

## 5. 클릭 수 설계

### 기존 흐름

```
카테고리 찾기 -> 펼치기 -> 값 입력 -> 메모 입력 -> 저장
```

### 새 흐름

```
미션 확인 -> 완료
```

필요할 때만:

```
미션 확인 -> +10분 -> 완료
```

실패 또는 변경:

```
미션 확인 -> 대체 수행 -> 대체 유형 선택
```

## 6. 시각 톤

현재의 다크 생산성 앱 톤은 유지하되, 카드 목록 느낌을 줄인다.

방향:

- 카드 나열보다 "판", "지도", "작전판"의 구성을 쓴다.
- 과한 판타지 장식은 피한다.
- 시즌 상태는 큰 숫자보다 작은 라벨, 진행선, 구역 상태로 표현한다.
- 색은 스탯별 의미 색을 유지하되 배경 전체가 한 색으로 치우치지 않게 한다.

금지:

- 일반 todo 체크리스트처럼 보이는 UI
- 카테고리 그리드가 첫 화면을 차지하는 구조
- XP 숫자만 커지는 보상 화면
- 설명문이 많은 온보딩

## 7. 게임적 피드백

### 미션 완료

즉시:

- 미션 카드가 clear 상태로 접힘
- 시즌 지도에서 오늘 노드가 점등
- XP는 보조 정보로 작게 표시

### 주간 작전 진전

- 구역 확보 게이지가 오른다.
- 위험 신호가 완화되거나 강화된다.
- 금요일 이후에는 남은 미션 수와 승리 가능성을 보여준다.

### 시즌 종료

- "성공/실패"보다 시즌 리포트를 우선한다.
- 어떤 규칙이 효과 있었는지 보여준다.
- 다음 시즌 후보 3개를 바로 제시한다.

## 8. Empty / Loading / Error

### Empty: 시즌 없음

첫 화면:

```
새 시즌을 시작한다
이번 달의 방향을 한 문장으로 적는다
[시즌 생성]
```

긴 설명 없이 바로 시즌 생성으로 들어간다.

### Loading

시즌 지도와 미션 카드의 skeleton을 보여준다. 빈 카드 5개를 반복하지 않는다.

### Error

오늘 미션 생성 실패:

- 기존 기록은 가능해야 한다.
- "기본 미션으로 열기" 버튼 제공.

## 9. 접근성

- `MissionDeck`의 완료 버튼은 키보드로 바로 접근 가능해야 한다.
- 시즌 지도는 색만으로 상태를 구분하지 않는다. `active`, `clear`, `risk` 텍스트 라벨 병행.
- 미션 완료 피드백은 `aria-live="polite"`로 전달한다.
- 애니메이션 감소 설정에서는 지도 점등과 카드 접힘을 즉시 처리한다.

## 10. Architect에게 넘길 요구

새 데이터 모델이 필요하다.

- `Season`
- `Challenge`
- `Episode`
- `Mission`
- `MissionCompletion`
- `PlanRecommendation`

API도 기존 `/api/categories`, `/api/logs`, `/api/stats`만으로는 부족하다.

필요한 초안:

- `GET /api/season/current`
- `POST /api/seasons`
- `POST /api/seasons/:id/generate-plan`
- `GET /api/missions/today`
- `POST /api/missions/:id/complete`
- `POST /api/missions/:id/substitute`
- `POST /api/episodes/:id/review`

Forge는 기존 `/today`를 바로 지우지 말고, 새 `SeasonCommand`를 루트에 배치한 뒤 기존 기록 UI를 `QuickLogDock`으로 낮추는 순서가 좋다.
